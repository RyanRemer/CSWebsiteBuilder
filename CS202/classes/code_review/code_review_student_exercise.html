In-Class Exercise

After the reading quiz, the your instructor will go through the Character.java file to demonstrate what to look for in a code review. After the demo is done, you will pair up with someone in the class and review some classes related to the one the you reviewed as a class. Your assignment is to submit, as a pair, a list of issues you find in the given classes. To receive full credit you must find at least 12 issues, and at least two issues in each class.

Be sure to include the name of both reviewers when you submit the assignment to LearningSuite. Both reviewers should individually submit the same document to LearningSuite. Submit your findings with the following format:

Class name

Line # - issue description

Example:

Sword

Line 4 - Missing constructor

Here are some tips:

  • Look for incorrect behavior
  • Getters or setters doing something suspicious
  • Loops not doing what they should
  • Missing documentation
  • Member variables that do not get set or used
  • Unused parameters in method calls
  • Misspellings
  • Poor naming
  • Inappropriate modifiers for methods or member variables
  • Missing member variables, methods, or constructors
